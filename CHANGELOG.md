# Changelog

## 1.3.0

- Added `Flex` and `Exact` matching modes
- Added whitespace-sensitive matching for spaces, tabs, indentation, and blank lines
- Preserved LF and CRLF compatibility in `Exact` mode
- Kept `Flex` as the default for backward compatibility
- Updated usage documentation

## 1.2.0

- Added direct AI code replacement in the active VS Code file
- Added automatic saving after successful replacements
- Added `All` and `1st` replacement modes
- Added persistent `Done` status
- Added configurable local port support
- Added light, dark, and automatic themes
- Added per-site controls
- Added experimental custom-site support
- Improved code matching and duplicate toolbar handling
